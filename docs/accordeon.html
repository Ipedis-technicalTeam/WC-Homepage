6 Bonnes Pratiques pour être en Conformité

Aujourd'hui, encore beaucoup de sites Web et d'applications mobiles sont conçus sans penser à la navigation des personnes en situation de handicap. Pourtant, pour ces personnes, l'outil digital représente un véritable levier d'intégration, et leur apporte bien souvent un surcroît d'indépendance. Selon les différents types de handicaps, les manquements les plus couramment relevés sur le Web ne sont pas les mêmes.

Comment ont été conçus les PDF ?

À l'origine, les documents PDF n'étaient pas conçus pour être lus via un ordinateur mais pour être imprimés. Aujourd'hui, les documents PDF sont de plus en plus utilisés par les entreprises pour échanger et diffuser de l'information numérique de façon plus professionnelle (présentation, sécurité pour la protection de données). Sans traitement, les documents PDF sont inaccessibles pour des personnes non-voyantes utilisant des outils tels qu'un lecteur d'écran.

Recommendation

Tab Container ‑ div
  • Add role="tablist" on div to indicate that the element serves as a container for a set of tabs.
  • Add a label (aria‑label or aria‑labelledby linked to a title) to the tab container describing its content.
    <div role="tabpanel" aria‑label="description of content">
Panel
  • Add role="tabpanel" on div containing tab contents to indicates the element serves as a container for tab panel content.
  • Add a unique identifier (id) to each panel.
  • Add an aria‑labelledby to each tab button and linked it to the id of corresponding button.
    <div id="tabpanel‑1" role="tabpanel" aria‑labelledby="tab‑1">
Tab buttons
  • Use <button> tag to define each button of the tab panel list.
  • Add role="tab" to each button.
  • Add a unique identifier (id) to each button.
  • Add aria‑controls to each button with corresponding panel id.
    <button id="tab‑1" type="button" role="tab" aria‑selected="true" aria‑controls="tabpanel‑1">
If panel is activated:
  • Add aria‑selected="true" to button of active panel to indicates the tab is selected and its associated tabpanel is displayed.
  • Add tabindex="0" to div containing panel contents of activated panel to add it to tab order.
    <div id="tabpanel‑1" role="tabpanel" tabindex="0" aria‑labelledby="tab‑1">
If panel is not activated
  • Add aria‑selected="false" to button to indicates the tab is not selected and its associated tabpanel is NOT displayed.
  • Add tabindex="‑1" to all unactive buttons to remove them from tab order.
    <button id="tab‑2" type="button" role="tab" aria‑selected="false" aria‑controls="tabpanel‑2" tabindex="‑1">

Keyboard Support

Tab
  • When focus moves into the tab list, places focus on the active tab element.
  • When the tab list contains the focus, moves focus to the next element in the tab sequence, which is the tabpanel element.
Right Arrow
  • Moves focus to the next tab.
  • If focus is on the last tab, moves focus to the first tab.
  • Activates the newly focused tab.
Left Arrow
  • Moves focus to the previous tab.
  • If focus is on the first tab, moves focus to the last tab.
  • Activates the newly focused tab.
Home
  • Moves focus to the first tab and activates it.
End
  • Moves focus to the last tab and activates it.

Complete W3C guidelines: https://w3c.github.io/aria‑practices/examples/tabs/tabs‑automatic.html
